# Release v1.12.0-beta

Fecha: 2026-05-12

Tipo de release: beta funcional del MVP.

## Resumen ejecutivo

`v1.12.0-beta` marca el primer release beta funcional de `gestor-seguros`. La aplicación ya permite cargar el Control Cartera desde Excel, consultar registros, buscar, revisar detalle, editar en memoria con validaciones, registrar cambios en bitácora de sesión, guardar una copia, guardar sobre el archivo cargado con respaldo automático, consultar ayuda integrada y mantener una preparación preliminar para empaquetado con PyInstaller.

Este release sigue siendo de ejecución mediante Python. No genera ejecutable final, instalador ni paquete portable para distribución.

## Funcionalidades incluidas

- Carga directa del Control Cartera desde `data/input/CONTROLCARTERA_V2.xlsx`.
- Selección manual de otros archivos `.xlsx`.
- Tabla de registros de solo lectura con búsqueda general y por columna.
- Vista de detalle del registro con doble clic.
- Edición controlada en memoria desde ventana modal.
- Validaciones bloqueantes y advertencias suaves durante la edición.
- Bitácora en memoria con cambios de la sesión.
- `Guardar como` para exportar una copia `.xlsx`.
- `Guardar` sobre el archivo cargado con respaldo automático previo.
- Ayuda integrada y ventana `Acerca de`.
- Atajos `F1`, `Ctrl+S` y `Ctrl+Shift+S`.
- Preparación preliminar para PyInstaller mediante `GestorSeguros.spec`.
- Modo técnico `python -m app --check`.

## Funcionalidades fuera de alcance

- Vencimientos.
- Avisos operativos.
- Generación DOCX.
- Dashboard.
- Exportaciones operativas adicionales.
- Reversión automática desde respaldos.
- Base de datos local.
- Integración con correo electrónico o WhatsApp.
- Instalador.
- Ejecutable final o release binario.

## Instalación

```powershell
python -m pip install -r requirements.txt
```

## Ejecución

Abrir la aplicación:

```powershell
python -m app
```

Chequeo técnico:

```powershell
python -m app --check
```

## Pruebas

```powershell
python -m pytest tests -p no:cacheprovider
```

## Advertencias sobre datos reales

- No se deben versionar archivos reales en `data/input/`, `data/output/` ni `data/backups/`.
- No se deben publicar capturas con información sensible.
- Las pruebas automatizadas y ejemplos usan datos ficticios.
- La aplicación puede visualizar datos reales solo en entorno local del usuario.
