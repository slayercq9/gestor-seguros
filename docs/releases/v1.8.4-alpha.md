# Release v1.8.4-alpha

Fecha: 2026-05-06

Tipo de release: tecnico inicial / alpha.

## Resumen ejecutivo

`v1.8.4-alpha` es el primer release tecnico inicial de `gestor-seguros`. Marca un punto revisable del proyecto donde la aplicacion ya cuenta con una GUI funcional en PySide6, lectura directa del Control Cartera operativo, visualizacion tabular de solo lectura, validaciones basicas de archivo, tema claro/oscuro persistente e identidad visual basica propia.

Este release no incluye ejecutable, instalador ni paquete portable. La ejecucion sigue siendo mediante Python.

## Funcionalidades incluidas

- GUI funcional con PySide6.
- Entrada principal con `python -m app`.
- Lectura directa del Control Cartera desde `data/input/CONTROLCARTERA_V2.xlsx`.
- Seleccion manual de otro archivo `.xlsx`.
- Validaciones basicas de ruta, extension y estructura esperada.
- Ventanas emergentes para errores reales.
- Tabla de registros en modo solo lectura.
- Resumen de carga con conteos y estado.
- Tema claro/oscuro con persistencia local mediante `QSettings`.
- Boton minimalista de tema con iconos `☀` / `🌙`.
- Icono propio en `assets/app_icon.svg`.
- Pruebas automatizadas con datos ficticios.

## Funcionalidades fuera de alcance

- No hay busqueda.
- No hay filtros.
- No hay edicion de registros.
- No hay guardado en Excel.
- No hay bitacoras funcionales.
- No hay persistencia SQLite operativa.
- No hay vencimientos automáticos.
- No hay generacion DOCX.
- No hay dashboards.
- No hay instalador.
- No hay ejecutable portable.
- No hay release binario.

## Instalacion

Requisitos:

- Python compatible con el proyecto.
- Dependencias definidas en `requirements.txt`.

Instalar dependencias:

```powershell
python -m pip install -r requirements.txt
```

## Ejecucion

Abrir la aplicacion:

```powershell
python -m app
```

Chequeo tecnico sin abrir la GUI:

```powershell
python -m app --check
```

## Pruebas

Ejecutar pruebas automatizadas:

```powershell
python -m pytest tests -p no:cacheprovider
```

## Advertencias sobre datos reales

- La base real es confidencial y no debe subirse a GitHub.
- No se deben versionar archivos en `data/input/`, `data/output/` ni `data/backups/`.
- No se deben publicar capturas con nombres, cedulas, polizas, placas, telefonos, correos, direcciones ni detalle real.
- Las pruebas y ejemplos deben usar datos ficticios o anonimizados.
- La aplicacion puede visualizar datos reales localmente, pero este release no debe incluirlos.
