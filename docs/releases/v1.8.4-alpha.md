# Release v1.8.4-alpha

Fecha: 2026-05-06

Tipo de release: técnico inicial / alpha.

## Resumen ejecutivo

`v1.8.4-alpha` es el primer release técnico inicial de `gestor-seguros`. Marca un punto revisable del proyecto donde la aplicación ya cuenta con una GUI funcional en PySide6, lectura directa del Control Cartera operativo, visualización tabular de solo lectura, validaciones básicas de archivo, tema claro/oscuro persistente e identidad visual básica propia.

Este release no incluye ejecutable, instalador ni paquete portable. La ejecución sigue siendo mediante Python.

## Funcionalidades incluidas

- GUI funcional con PySide6.
- Entrada principal con `python -m app`.
- Lectura directa del Control Cartera desde `data/input/CONTROLCARTERA_V2.xlsx`.
- Selección manual de otro archivo `.xlsx`.
- Validaciones básicas de ruta, extensión y estructura esperada.
- Ventanas emergentes para errores reales.
- Tabla de registros en modo solo lectura.
- Resumen de carga con conteos y estado.
- Tema claro/oscuro con persistencia local mediante `QSettings`.
- Botón minimalista de tema con íconos `☀` / `🌙`.
- Ícono propio en `assets/app_icon.svg`.
- Pruebas automatizadas con datos ficticios.

## Funcionalidades fuera de alcance

- No hay búsqueda.
- No hay filtros.
- No hay edición de registros.
- No hay guardado en Excel.
- No hay bitácoras funcionales.
- No hay persistencia SQLite operativa.
- No hay vencimientos automáticos.
- No hay generación DOCX.
- No hay dashboards.
- No hay instalador.
- No hay ejecutable portable.
- No hay release binario.
- No incluye datos reales.

## Instalación

Requisitos:

- Python compatible con el proyecto.
- Dependencias definidas en `requirements.txt`.

Instalar dependencias:

```powershell
python -m pip install -r requirements.txt
```

## Ejecución

Abrir la aplicación:

```powershell
python -m app
```

Chequeo técnico sin abrir la GUI:

```powershell
python -m app --check
```

## Pruebas

Ejecutar pruebas automatizadas:

```powershell
python -m pytest tests -p no:cacheprovider
```

## Advertencias sobre datos reales

- La base real es confidencial y no debe subirse a GitHub.
- No se deben versionar archivos en `data/input/`, `data/output/` ni `data/backups/`.
- No se deben publicar capturas con nombres, cédulas, pólizas, placas, teléfonos, correos, direcciones ni detalle real.
- Las pruebas y ejemplos deben usar datos ficticios o anonimizados.
- La aplicación puede visualizar datos reales localmente, pero este release no debe incluirlos.
